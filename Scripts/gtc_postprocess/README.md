# Smart Sharpen (post-process)

A lightweight, drop-in screen-space sharpening effect — recovers detail lost to
TAA/TSR blur, works at any resolution, and can be tuned globally or per-area.
Mirrors the feature set of the marketplace "Smart Sharpen" effect, built native.

## Pieces

- **`Source/GTC_UE5/Camera/SmartSharpen.{h,cpp}`** — `FSmartSharpen`, the pure
  per-pixel sharpening math (distance falloff, shadow-safe luminance gate,
  foliage blend, local/global combine, resolution-independent texel step). Unit
  tested in `Camera/Tests/SmartSharpenTest.cpp` (`GTC.Camera.SmartSharpen.*`).
  This is the parity oracle the material's HLSL mirrors.
- **`Source/GTC_UE5/Camera/GTCSmartSharpen.{h,cpp}`** — `AGTCSmartSharpen`, the
  drop-in actor. One `UPostProcessComponent` + a dynamic instance of the sharpen
  material; no tick. Editor billboard + text show the live amount at a glance.
- **`build_smart_sharpen_material.py`** — authors `M_GTC_SmartSharpen`, the
  post-process material the actor drives.

## Apply the material (one-time, dedicated editor session)

Authoring a Post-Process-domain material from a busy editor can deadlock the
game thread (it recursively loads `DefaultPostProcessMaterial`), so do it in a
quiet session:

1. Launch the GTC editor (`Launch-GTC-with-MCP.command`).
2. `Tools ▸ Execute Python Script…` → `build_smart_sharpen_material.py`
   (or paste into the Python console and press Enter twice).
3. Wait for `OK: M_GTC_SmartSharpen built + saved`. Shader compile can take ~1
   min. If it says the asset already exists, delete it in the Content Browser
   and re-run.

`M_GTC_SmartSharpen` lives at `/Game/Materials/PostProcess/` — the actor's
default `SharpenMaterial` points there, so once it exists the effect is live.

The material is generated by this script (no `.uasset` is committed — it is
project-authored/original, no provenance ledger row needed).

## Use it

Drag `AGTCSmartSharpen` into a level. Defaults give a global, whole-scene
sharpen at amount 1.0. Key knobs (Details panel, "Smart Sharpen"):

- **SharpenAmount / PixelRadius** — strength and kernel reach (px, res-independent).
- **ShadowFloor** — luminance below which sharpening fades out (shadow safety).
- **Distance** — `bDistanceControl` + Near/Far amounts and cm band.
- **Foliage** — `bFoliageControl` + `FoliageAmount` (foliage = custom-stencil tagged).
- **Influence** — `bUnbound` (global) vs a bounded local volume with `BlendRadius`,
  plus `BlendWeight`/`Priority` for stacking multiple actors.

Works with TSR/TAA/FXAA/SMAA and both deferred and forward rendering (it is a
normal after-tonemapping post-process pass).
